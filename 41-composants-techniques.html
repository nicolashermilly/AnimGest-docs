DocumentActions NEW S38

Composant unique PDF / Email / Signature — réutilisable sur tous les écrans

DocumentActions est le composant central de toutes les actions documentaires. Il expose 3 boutons : aperçu PDF (modal iframe), envoi email (modal avec sélecteur contact), envoi à la signature Yousign. Obligatoire sur tous les écrans gérant des documents : devis, factures, avoirs, achats, compta.

💡 Règle d'or : ne jamais recoder ces 3 boutons individuellement. Toujours utiliser DocumentActions.

Import

import { DocumentActions } from "@/components/DocumentActions";

Usage minimal

<DocumentActions docType="DEVIS" docId={dce.DCE_ID} docNum={dce.DCE_NUMERO} />

Usage complet

<DocumentActions
  docType="FACTURE"       // DEVIS | FACTURE | AVOIR | ACHAT | COMPTA
  docId={fac.FAC_ID}
  docNum={fac.FAC_NUM}
  tieId={fac.TIE_ID}      // charge contacts via GET /api/contacts?tieId=...
  clientNom={tie.TIE_RAISON_SOCIALE}
  clientEmail={tie.TIE_EMAIL}
  clientTel={tie.TIE_TEL}
  size="md"            // "sm" icônes seules | "md" icônes + labels
/>

Props

PropTypeReq.Description

docIdnumber | string✓Identifiant BDD du document
docNumstring✓Numéro affiché (DCE-2603-0026…)

Comportement des 3 boutons

🖨️ PDF

Modal avec iframe chargée depuis l'URL PDF. Boutons "Ouvrir dans un onglet ↗" et "Imprimer". Jamais de navigation nouvelle page.

📧 Email

Modal avec sélecteur de destinataire (client + contacts), objet et message pré-remplis. La sélection met à jour Nom, Prénom, Email, Téléphone.

🖊️ Signature

Ouvre SignatureModal avec prefill. Génère PDF via pdfkit, upload Yousign, crée le signataire. TTS vocal à chaque étape.

🔗 Sélecteur contact commun

Email et Signature partagent la même logique : GET /api/contacts?tieId=... — dropdown client principal + contacts du dossier.

Règle modale obligatoire

✓ Toutes les éditions PDF s'ouvrent dans une modal avec iframe
✗ Ne jamais naviguer vers une nouvelle page pour afficher un PDF
✗ Ne jamais utiliser window.open() pour les PDFs

SignatureModal NEW S38

Signature électronique Yousign avec TTS et sélecteur de contact

Modal dédiée à l'envoi de documents à la signature via l'API Yousign. Intègre une lecture vocale TTS (Web Speech API, voix française), un sélecteur de contact et une barre de progression animée.

Flux technique

Formulaire signataire
→
POST /devis/:id/signature
→
Génération PDF (pdfkit)
→
Upload PDF Yousign
→
Création signataire
→
BDD signature_demande ✓

Étapes TTS

  • Étape 1 : "Formulaire de signature pour le devis [num]. Renseignez les informations du signataire."
  • Étape 2 : "Génération du PDF et envoi à Yousign en cours. Veuillez patienter."
  • Étape 3 : "Demande de signature créée avec succès."
  • Bouton 🔊 pour relire l'étape courante.

Configuration BDD

-- Table param_yousign
UPDATE param_yousign SET
  YS_API_KEY  = 'sk-sandbox-...',
  YS_ENV      = 'sandbox',    -- ou 'production'
  YS_BASE_URL = 'https://api-sandbox.yousign.app/v3',
  YS_ACTIF    = 1
WHERE ID = 1;
⚠️ URL interne Docker : l'API génère le PDF via http://micro_nginx/api/devis/:id/pdf (nom du service nginx, pas localhost).

Charte graphique CSS Variables

Paramétrable depuis Mes paramètres → Charte graphique

Toutes les pages utilisent exclusivement les variables CSS de la charte. Jamais de couleurs hardcodées dans le code.

Variables CSS

--accentCouleur principale (#4F46E5)
--surfaceFond des cards et sections
--surface-pageFond de page
--surface-hoverFond au survol des éléments
--border-colorCouleur des bordures
--text-primaryTexte principal
--text-secondaryTexte secondaire
--text-mutedTexte atténué

Classes utilitaires

.cardCard avec bordure et fond surface
.btn-primaryBouton principal (accent)
.btn-ghostBouton secondaire transparent
.inputChamp de saisie
.labelLabel de formulaire
.space-y-6Espacement standard entre sections

Pattern page standard

export default function MaPage() {
  return (
    <div className="space-y-6">
      <PageHeader title="..." subtitle="..." />
      <div className="border-b border-[var(--border-color)]">
        <button className="border-b-2 border-[var(--accent)] text-[var(--accent)]">Onglet actif</button>
      </div>
      <div className="card p-5">
        {/* Contenu */}
      </div>
    </div>
  );
}
✅ Correct : text-[var(--accent)], bg-[var(--surface)], border-[var(--border-color)]
✗ Interdit : style={{ background: '#0f0f0f' }}, font-family: Georgia, classes hardcodées bg-gray-900

Page Interfaces API NEW S38

/app/parametres/api-keys — Configuration des services externes

Page accordéon de configuration de toutes les intégrations externes. Chaque section gère sa propre table BDD dédiée. Toggle actif/inactif par service.

🖊️ Yousign

Clé API, env sandbox/prod, URL, secret webhook. Table : param_yousign

📡 Pennylane (PDP)

URL API, SIREN, TVA intracom, env TEST/PROD. Table : param_pdp

📧 SMTP

Hôte, port, utilisateur, mot de passe, TLS. Table : param_smtp

💳 Stripe

Clé publique, secrète, webhook, environnement. Table : param_stripe

💡 Pour ajouter une intégration : créer param_xxx, ajouter GET/POST /param-xxx dans custom-routes.ts, et ajouter l'entrée dans API_SECTIONS de la page.
