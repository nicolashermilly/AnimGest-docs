SignatureModal
Signature électronique Yousign avec TTS et sélecteur de contact

Flux technique

Formulaire signataire → POST /devis/:id/signature → Génération PDF (pdfkit) → Upload PDF Yousign → Création signataire → BDD signature_demande ✓

Configuration BDD requise

-- Table param_yousign (GET/POST /api/param-yousign)
UPDATE param_yousign SET
  YS_API_KEY  = 'sk-sandbox-...',
  YS_ENV      = 'sandbox',
  YS_BASE_URL = 'https://api-sandbox.yousign.app/v3',
  YS_ACTIF    = 1
WHERE ID = 1;
⚠️ URL interne Docker : l'API génère le PDF via http://micro_nginx/api/devis/:id/pdf (nom du service nginx, pas localhost).
💡 Restriction S41 : le bouton Signature n'est affiché que pour docType="DEVIS". Les factures ne passent pas par Yousign.

Architecture custom-routes.ts
Règles critiques — fichier compilé dans l'image NestJS

Règles immuables

✗ inject_v4_final.js — NE JAMAIS UTILISER (catch orphelin → SyntaxError crash loop)
✓ Toutes les routes custom vivent dans api/src/custom-routes.ts, compilées dans l'image Docker
⚠️ docker compose restart efface les fichiers injectés manuellement — toujours passer par le build

Pattern body parsing obligatoire

NestJS ne parse pas le body dans instance.use(). Toujours utiliser ce pattern pour les POST/PUT/PATCH :

// CORRECT — pour toutes les routes POST dans instance.use()
const b: any = await new Promise<any>((resolve) => {
  let s = '';
  req.on('data', (chunk: any) => { s += chunk; });
  req.on('end', () => {
    try { resolve(JSON.parse(s || '{}')); } catch { resolve({}); }
  });
});

// INTERDIT — req.body est toujours undefined dans instance.use()
const b = req.body || {}; // ❌ ne jamais faire ça

Pattern TypeScript obligatoire

// Toujours typer les paramètres des callbacks
instance.use('/ma-route', async (req: any, res: any, next: any) => {
  // ...
});

// b doit toujours être typé any
const b: any = await new Promise<any>(...);

Nginx — préfixe /api/

Nginx strip le préfixe /api/ avant de transmettre à l'API. Les routes dans custom-routes.ts s'enregistrent sans /api/. Ex: instance.use('/reglements') répond à /api/reglements.

cd "$root"

# 2. Build
cd deploy

# Frontend seul
docker compose build frontend --no-cache
docker compose up -d frontend
docker compose restart nginx

# API seule
docker compose build api --no-cache
docker compose up -d api

# Les deux
docker compose build api frontend --no-cache
docker compose up -d api frontend
docker compose restart nginx
