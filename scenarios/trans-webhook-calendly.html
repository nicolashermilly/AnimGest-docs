Webhook Calendly

Integration automatique des reservations en ligne — S83

Principe

Lorsqu'un client reserve un creneau sur Calendly, Calendly envoie automatiquement un webhook HTTP POST vers Anim'Gest. Le systeme cree alors une seance en base de donnees sans intervention manuelle.

Client reserve sur Calendly
→
POST /api/webhooks/calendly
→
Seance creee en BDD

Configuration dans Calendly

1
Copier l'URL Webhook
Dans Anim'Gest : Parametres > Calendly > encart "URL Webhook Calendly"
Cliquer sur "Copier" pour copier l'URL.
2
Dans Calendly.com
Settings > Integrations > Webhooks > + New Webhook
3
Configurer le webhook
URL : coller l'URL copiee depuis Anim'Gest
Evenement a ecouter : invitee.created
Scope : Organization ou User selon votre compte
4
Tester
Faire une reservation test sur Calendly — verifier dans Anim'Gest > Seances qu'une nouvelle seance apparait.
Important : Le webhook Calendly necessite que votre serveur Anim'Gest soit accessible depuis Internet (URL publique). En environnement local (localhost), utiliser un tunnel comme ngrok pour les tests.

Mapping types de RDV

Le systeme associe automatiquement le slug Calendly au type de seance Anim'Gest.

Slug CalendlyType de seanceMetier
ec-1h30Bilan ComportementalComportementaliste
suivi-1hSeance de suiviComportementaliste
educ-coursCours individuel educationEducateur
toilettageToilettageToiletteur
garde-visiteVisite a domicilePet-Sitter
garde-pensionGarde / PensionPet-Sitter
cours-equitCours equitation individuelCentre Equestre
balade-equitBalade equestreCentre Equestre
consult-elevSuivi portee / elevageEleveur
Le slug correspond au nom de l'event type dans l'URL Calendly (ex: calendly.com/votre-nom/ec-1h30). Nommez vos event types en respectant ces slugs pour que le mapping soit automatique.

Comportement du systeme

SituationComportement
Client reconnu par emailSeance liee au client existant (TIE_ID)
Client reconnu par nomSeance liee au client (recherche approx.)
Client inconnuSeance creee sans lien client (a completer manuellement)
Type RDV reconnu (slug)STY_ID associe automatiquement
Type RDV inconnuSeance creee sans type (a renseigner)
Evenement autre que invitee.createdIgnore (reponse 200 OK)

Toutes les seances creees par webhook sont en statut PLANIFIEE et FACTURABLE par defaut. Un log est enregistre dans param_config (groupe CALENDLY_LOG) pour chaque appel recu.

Route API

POST /api/webhooks/calendly
Content-Type: application/json

// Payload Calendly (invitee.created)
{
  "event": "invitee.created",
  "payload": {
    "event": {
      "start_time": "2026-05-03T10:00:00Z",
      "end_time":   "2026-05-03T11:30:00Z",
      "location":   { "location": "Cabinet Paris 15" }
    },
    "invitee": {
      "name":  "Marie Dupont",
      "email": "marie@exemple.fr"
    },
    "event_type": {
      "name": "Bilan comportemental 1h30",
      "slug": "ec-1h30"
    }
  }
}

// Reponse OK
{ "ok": true, "created": true }

// Reponse evenement ignore
{ "ok": true, "skipped": true }
