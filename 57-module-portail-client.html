Portail Client v2

Espace web sécurisé permettant au client d'accéder à ses séances, son suivi CAC, ses documents et de confirmer ses rendez-vous — sans compte Anim'Gest.

01 Vision & périmètre
📅
Séances
Consulter ses séances passées et à venir, confirmer ou annuler un rendez-vous directement depuis le portail.
📈
Suivi CAC
Accéder à l'évolution des 14 critères comportementaux de ses animaux, graphique de progression sprint par sprint.
📄
Documents
Télécharger devis, factures, signer électroniquement via Yousign, payer en ligne via Stripe.
🔒
Accès sécurisé
Lien unique par client, token UUID, session 24h, historique des connexions, révocation possible à tout moment.
ℹ️
Ce module remplace et enrichit la page /client/[token] existante qui gère uniquement la consultation de devis. La nouvelle version conserve la même URL mais ajoute les onglets Séances, Animaux et Documents.
02 Flux d'accès client
1
Praticien crée l'accès dans /parametres/portail
2
Email auto avec lien unique /client/UUID
3
Client clique → session créée (24h)
4
Dashboard personnalisé affiché
5
Connexion enregistrée en BDD
03 Base de données — 4 tables BDD
TableRôleColonnes clés
portail_client_acces Accès sécurisé d'un client (un token par client) PCA_TOKEN, TIE_ID, PCA_ACTIF, PCA_DATE_EXPIRATION, PCA_NB_CONNEXIONS
portail_client_session Sessions actives (audit, sécurité) PCS_SESSION_TOKEN, PCA_ID, PCS_IP, PCS_DATE_DEBUT, PCS_ACTIF
portail_client_notification Historique emails/SMS envoyés PCN_TYPE, PCN_CANAL, PCN_STATUT, TIE_ID, SEA_ID, DCE_ID, FAC_ID
portail_client_confirmation_seance Réponses client aux convocations PCC_REPONSE (CONFIRME/ANNULE/REPORTE), SEA_ID, PCC_MESSAGE, PCC_VU_PRATICIEN
⚠️
Le script SQL portail_client_tables.sql est disponible dans les livrables S61. À exécuter dans HeidiSQL sur micro_logiciel avant le déploiement des routes API.

ℹ️
Toutes les routes /portail/ utilisent un middleware de vérification du token de session (PCS_SESSION_TOKEN + PCS_ACTIF=1 + PCA_ACTIF=1). Les routes /portail-admin/ nécessitent une session Anim'Gest praticien normale.
05 Pages Frontend UI
🌐
/client/[token] — Portail client
Page publique accessible sans login Anim'Gest. Onglets : Tableau de bord · Séances · Mes animaux · Devis & Factures. Session stockée en sessionStorage.
⚙️
/app/parametres/portail — Gestion accès
Interface praticien pour créer/révoquer les accès clients. Liste des clients avec statut accès, bouton Créer accès, historique connexions.
06 Règles de sécurité
RègleDétail
Stockage sessionsessionStorage uniquement — jamais localStorage (effacé à fermeture onglet)
Expiration session24h maximum depuis PCS_DATE_DEBUT
Token accèsUUID v4 généré côté serveur, stocké en SHA256 dans PCA_TOKEN
RévocationPCA_ACTIF = 0 révoque immédiatement toutes les sessions actives
CAC portailLecture seule — jamais de modification depuis le portail client
DocumentsSeuls les documents liés au TIE_ID du token sont accessibles
Confirmation séancePCC_VU_PRATICIEN alerte le praticien dans son interface
